# HDB Resale Flat Prices — ETL Pipeline (Part 1)

Senior Data Engineer Technical Test — Part 1: Developing Data Pipelines.

## Project structure

```
hdb_etl/
├── README.md                       <- this file
├── requirements.txt
├── config.py                       <- all paths, dataset ids, whitelists, thresholds
├── run_pipeline.py                 <- CLI entrypoint: runs the whole ETL end-to-end
├── src/
│   ├── extract.py                  <- programmatic data.gov.sg download + combine
│   ├── profile.py                  <- data profiling (completeness, frequency, stats)
│   ├── validate.py                 <- field validation rules (month/town/flat_type/flat_model/storey_range)
│   ├── clean.py                    <- remaining lease, dedup, price anomaly flagging
│   ├── transform.py                <- resale_identifier construction
│   └── hash_utils.py               <- irreversible hashing (SHA-256)
├── notebooks/
│   └── ETL_Pipeline.ipynb          <- narrated, executed walkthrough (the main deliverable)
├── tests/
│   └── generate_sample_data.py     <- schema-accurate synthetic data for offline/sandboxed runs
├── docs/
│   └── ASSUMPTIONS.md              <- every heuristic/assumption, documented per requirement
└── data/
    ├── raw/                        <- untouched source files, as downloaded
    ├── cleaned/                    <- records passing all data-quality rules
    ├── failed/                     <- records rejected at any stage + reason
    ├── transformed/                <- cleaned data + resale_identifier
    └── hashed/                     <- cleaned data + hashed resale_identifier
```

## How to run

```bash
pip install -r requirements.txt

# Real run: pulls the actual data.gov.sg files (requires outbound internet access)
python run_pipeline.py

# Offline / sandboxed demo: uses a generated, schema-accurate synthetic dataset instead
python run_pipeline.py --use-sample-data

# Force re-download of raw files even if a local copy already exists in data/raw/
python run_pipeline.py --force-download
```

Or open `notebooks/ETL_Pipeline.ipynb` for the fully narrated, cell-by-cell walkthrough
(this notebook has already been executed once end-to-end against the sample dataset, so its
saved outputs can be reviewed without re-running anything).

## What each Data Quality / Transformation requirement maps to

| Assignment requirement | Implementation |
|---|---|
| Combine datasets into one master dataset | `src/extract.combine_master_dataset` (outer-join concat) — every file in the collection is discovered dynamically (`discover_collection_dataset_ids`) and fetched, none are pre-selected |
| Data profiling | `src/profile.py` |
| Validate month/town/flat_type/flat_model/storey_range | `src/validate.py` — town/flat_type/flat_model all use the same purely statistical, dataset-derived rule (`validate_categorical_by_rarity`); no hardcoded whitelists |
| Remaining lease (99y, floored) | `src/clean.compute_remaining_lease` |
| Duplicate composite key → keep higher price | `src/clean.resolve_duplicate_keys` |
| Anomalous resale price heuristic | `src/clean.flag_anomalous_prices` (documented in `docs/ASSUMPTIONS.md`) |
| `resale_identifier` transformation | `src/transform.build_resale_identifier` |
| Duplicate check post-transform | `transform.resolve_duplicate_transformed_rows` |
| Irreversible hash preserving uniqueness | `src/hash_utils.py` (SHA-256, rationale in module docstring) |
| Raw / Cleaned / Transformed / Failed / Hashed outputs | `run_pipeline.py`, written to `data/*` |

All heuristics, thresholds, and judgement calls are documented in **`docs/ASSUMPTIONS.md`**.

## Note on network access

`src/extract.py` calls data.gov.sg's Open Data API directly (no manual downloads, no manual
row/column edits to the source files). If you're running this in an environment without
outbound internet access to `data.gov.sg`, use the `--use-sample-data` flag (or set
`USE_SAMPLE_DATA = True` in the notebook) to exercise the complete pipeline — extraction
through hashing — against a small, schema-accurate synthetic dataset generated by
`tests/generate_sample_data.py`. The pipeline logic itself is identical either way; only the
extraction source changes.
